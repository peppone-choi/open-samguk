# 테스트 정책

본 프로젝트는 결정론적 턴 처리와 메모리 상태 기반 아키텍처를 전제로 합니다. 테스트는 동일 입력에서 동일 결과를 재현하는지 검증하는 데 초점을 둡니다.

## 테스트 레이어

- 단위 테스트: 계산 로직, 제약 검사, RNG 유틸
- 통합 테스트: 턴 처리 파이프라인, 이벤트/경제 처리
- 복구 테스트: 스냅샷/저널 적재 후 상태 일치
- 실시간 테스트: WS/SSE 이벤트 순서와 페이로드
- 부하 테스트: 대규모 장수/도시 상태에서 처리 지연 검증

## 결정론 검증

- RNG 시드는 명시적으로 관리 (LiteHashDRBG 사용)
- 모든 게임 로직(전투, 내정, 이벤트)은 주입된 RNG 인스턴스를 사용
- 동일 입력/시드에 대해 결과 해시 비교
- 전투/경제/이벤트 결과 재현성 확인

## 복구 시나리오

- 스냅샷 + 저널 재생 후 상태 비교
- 비정상 종료 후 재시작 테스트
- 저널 손상 시 복구 실패 처리 정책 검증

## 회귀 테스트

- 레거시 결과와의 비교 테스트
- 핵심 커맨드 결과 스냅샷 비교
- 월간 처리 결과(경제/이벤트) 비교

## 테스트 데이터 정책

- 레거시 샘플 데이터는 스냅샷으로 제공
- 테스트 케이스는 고정 입력을 사용
- 결과 로그는 JSON으로 저장
