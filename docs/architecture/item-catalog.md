# 아이템 카탈로그 (Item Catalog)

레거시 아이템 시스템 분석 및 `packages/logic`에 구현해야 할 아이템 효과 목록입니다.

## 1. 시스템 개요

### 1.1 클래스 계층 구조

```
iAction (인터페이스)
├── DefaultAction (trait - 기본 구현)
└── BaseItem (기본 아이템 클래스)
    └── BaseStatItem (스탯 아이템 - 명마/무기/서적)
```

### 1.2 레거시 파일 위치

| 경로                                 | 설명                |
| ------------------------------------ | ------------------- |
| `legacy/hwe/sammo/iAction.php`       | 액션 인터페이스     |
| `legacy/hwe/sammo/DefaultAction.php` | 기본 구현 trait     |
| `legacy/hwe/sammo/BaseItem.php`      | 아이템 기본 클래스  |
| `legacy/hwe/sammo/BaseStatItem.php`  | 스탯 아이템 클래스  |
| `legacy/hwe/sammo/ActionItem/*.php`  | 개별 아이템 (161개) |

### 1.3 아이템 속성

| 속성         | 타입   | 설명                  |
| ------------ | ------ | --------------------- |
| `rawName`    | string | 아이템 원본 이름      |
| `name`       | string | 표시 이름 (효과 포함) |
| `info`       | string | 효과 설명 (HTML)      |
| `cost`       | int    | 가격                  |
| `consumable` | bool   | 소모품 여부           |
| `buyable`    | bool   | 구매 가능 여부        |
| `reqSecu`    | int    | 치안 요구치           |

## 5. 경매 시스템

### 5.1 경매 클래스

| 클래스              | 설명               |
| ------------------- | ------------------ |
| `Auction`           | 경매 추상 클래스   |
| `AuctionUniqueItem` | 유니크 아이템 경매 |
| `AuctionBuyRice`    | 쌀 구매 경매       |
| `AuctionSellRice`   | 쌀 판매 경매       |

### 5.2 경매 규칙

- **입찰 통화**: 유산 포인트(inheritancePoint) 또는 금/쌀
- **최소 입찰액**: `GameConst::$inheritItemUniqueMinPoint`
- **입찰 상회**: 최소 1% 또는 10포인트 이상 높게
- **시간 연장**: 입찰 시 자동 연장 (턴 시간 비례)
- **낙찰 제한**: 동일 부위 유니크 보유 시 입찰 불가

### 5.3 경매 가능 여부

```typescript
// isBuyable() === false인 아이템만 경매 가능
if (item.isBuyable()) {
  return "구매할 수 있는 아이템입니다.";
}
```

## 6. 포팅 시 고려사항

### 6.1 스탯 시스템

- `onCalcStat` 훅으로 스탯 계산 파이프라인 구현
- 복수 아이템/특기 효과의 순차 적용
- 디버프는 `onCalcOpposeStat`으로 분리

### 6.2 트리거 시스템

- `GeneralTrigger`: 턴 단위 처리
- `WarUnitTrigger`: 전투 페이즈 단위 처리
- 트리거 ID로 중복 발동 방지 (`TYPE_DEDUP_TYPE_BASE`)

### 6.3 소모품 처리

- `tryConsumeNow()` 메서드로 소모 체크
- `REMAIN_KEY` 로 잔여 횟수 저장
- `onArbitraryAction(actionType='장비매매', phase='구매')` 로 초기화

### 6.4 유니크 제한

- `GameConst::$maxUniqueItemLimit` - 연차별 유니크 보유 제한
- `GameConst::$allItems` - 아이템 타입별 점유 수량 제한
