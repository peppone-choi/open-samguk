# 코드 리팩토링

$ARGUMENTS 파일의 코드를 Clean Code 원칙에 따라 리팩토링합니다.

## 수행 작업
1. 코드 분석 및 개선점 식별
2. Clean Code 원칙 적용:
   - 단일 책임 원칙 (SRP)
   - 명확한 네이밍
   - 함수 크기 최소화
   - 중복 제거
3. 리팩토링 실행
4. `pnpm typecheck && pnpm test` 실행하여 기능 검증
5. 실패 시 수정 후 재검증 (성공할 때까지 반복)

## 주의사항
- 기존 동작을 변경하지 않음
- 테스트가 없으면 먼저 테스트 작성
- any 타입 제거
