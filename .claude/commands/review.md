# 코드 리뷰

$ARGUMENTS 코드를 시니어 개발자 관점에서 비판적으로 리뷰합니다.

## 리뷰 관점

1. **보안 취약점**
   - SQL Injection
   - XSS
   - 인증/인가 문제

2. **성능 문제**
   - N+1 쿼리
   - 불필요한 연산
   - 메모리 누수

3. **코드 품질**
   - 타입 안전성
   - 에러 처리
   - 테스트 커버리지

4. **아키텍처**
   - 단일 책임 원칙
   - 의존성 방향
   - 결합도/응집도

## 출력 형식

```markdown
### Critical (즉시 수정 필요)

- ...

### Warning (권장 수정)

- ...

### Suggestion (개선 제안)

- ...
```
